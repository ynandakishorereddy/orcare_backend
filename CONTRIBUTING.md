# Contributing to ORCare Backend

First off, thank you for considering contributing to ORCare! It's people like you that make open-source software such a great community.

## How to Contribute

### 1. Reporting Bugs
If you find a bug, please create an issue on GitHub. Include:
*   A clear and descriptive title.
*   Steps to reproduce the issue.
*   Expected and actual results.
*   Any relevant logs or screenshots.

### 2. Suggesting Enhancements
We welcome ideas for new features! Create an issue describing your idea, why it would be useful, and how you envision it working.

### 3. Pull Requests
1.  Fork the repository.
2.  Create a new branch for your feature or bug fix (`git checkout -b feature/your-feature-name`).
3.  Make your changes.
4.  Ensure the code builds correctly and any tests pass.
5.  Commit your changes with clear, descriptive commit messages (`git commit -m "feat: add new feature"`).
6.  Push your branch to your fork (`git push origin feature/your-feature-name`).
7.  Open a Pull Request against the `main` branch of this repository.

### Coding Style
*   Follow standard Node.js/Express.js best practices.
*   Keep files small and focused (e.g., separate routes from controllers).
*   Add JSDoc comments for complex functions.
